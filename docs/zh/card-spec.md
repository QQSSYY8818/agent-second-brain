# 卡片规格

*[English](../en/card-spec.md)*

一张卡 = 一个 Markdown 文件 = 一个不可再分的判断单元。文件名即稳定 ID，**永不改名**；禁字符 `\/:*?"<>|` 与尾随空格。

## 1. Frontmatter（必填十字段）

```yaml
---
schema: 1
type: 概念            # 见 §2 卡型表
tier: T1              # T0 登记 / T1 标准卡 / T2 精卡 / T3 承重卡
projects: [00]        # 归属项目编号（可多个）
confidence: 已验证    # 已验证（自证/实测）/ 文献（外部来源）/ 联想（未核实推断）
status: 已审          # 草稿 / 已审 / 已取代（已取代必带 取代者:: 边）
aliases: ["中文别名", "english alias"]   # T1+ 至少 2 个，含英文（检索命中率的关键）
origin: 2026-07-11    # 知识产生日 ≤ created ≤ updated
created: 2026-07-11
updated: 2026-07-11
source: 来源路径或描述
---
```

可选字段：`used`/`last_hit`（使用计数，脚本维护）、`活性`（`显性` 或 `辅助 YYYY-MM-DD`，见 architecture.md §4）、`q_status`（问题卡专用：open/partial/answered）、`provenance`（概念卡必填：文献共识/文献+自研/自研数据/自研推导）。

## 2. 卡型（14 类，文件名前缀=类型）

| 前缀 | 用途 |
|------|------|
| `概念-` | 一个经过判断的结论/机制/设计（必带 provenance） |
| `文献-` | 一篇外部文献的要点与判断 |
| `方法-` | 可复用的操作序列/管线/SOP 摘要 |
| `事例-` | 一场完成战役的复盘（目的/路径/产出/可借鉴） |
| `对话-` | 一次关键对话的决策现场 |
| `问题-` | 开放问题（q_status 跟踪；answered 必带指向答案卡的 支持:: 边） |
| `知识点-` | 维基式定义（环外知识平面，散装，被环内卡按关键词引用） |
| `符号-` | 一符一卡（公式符号的定义与易混淆项） |
| `关联-` | 纯联想级连接的载体（confidence: 联想） |
| `清单-` | 线性末端资源列表（直达链接义务：本地文件带 file:/// 一键可达） |
| `编年-` | 时间轴事件底账（一行一事，供"做过类似的事吗"回溯） |
| `枢纽-` | 每项目一张：一句话定位 + 现况节（带最后核实日期）+ 开放问题 |
| `MOC-` | 主题地图：结论摘要节（5-10 行合订）+ 分区链接 |
| `登记-` | T0 文件登记（万级，只进登记目录不进检索主入口） |

## 3. 边（关联的一等公民）

写在卡正文 `## 关联` 节，**禁裸链接**，每条必带一行理由：

```
边型:: [[目标卡名]] — 一行理由（为什么连）
```

十种边型：`支持` `冲突` `延伸` `类比` `上位` `用法` `用材` `同理` `出处` `引用`（另有 `取代者` 专用于已取代卡）。

- **强弱标注**：理由可前缀 `（强）`=功能依赖/同一机制、`（弱）`=同域类比/潜在互益，不标=中性。检索先走强边。
- **上位轴**：实例卡指向家族概念卡 `上位:: [[概念-X家族]] — 本卡是 X 在<场景>的实例`；下位不设边型，一律邻接表反查入边。家族抽象必须一句话可陈述，禁为连而连。
- **强边回写**：建立 冲突/取代/支持 边时，必回写目标旧卡一行反向语境并刷其 updated；弱边不回写。
- **自动成边只限 出处/引用**（机械可证），其余边型一律提名制——由人（或主 agent）确认。

## 4. 写卡纪律

- **description 结论前置**：正文首个叙述句的前 40 字必须自含判别力——先写"结论是什么"，再写限定与背景。目录 description 取自这句，它决定注入与 grep 的命中质量。
- **公式白话义务**：每个公式必须紧跟一句白话解读（「——式子在说：…」），禁裸公式。
- **建卡仪式=反向检查**：先 grep 总目录+扫相关 MOC，问"与哪张卡冲突/重复？"
- **门槛两问**（T1+）：三个月后还用吗？换个项目还成立吗？不满足→降为索引行。
- **零孤儿原则**：非枢纽/MOC 卡必须至少有一条边（lint L10 强制）。
- **忙时降级**：来不及写卡就在 `_inbox/` 记一行速记（标题+一句话）——合规的最低动作。
- 子代理产的卡 status 只能=`草稿`；仅 `已审` 可挂 MOC/HOME。
- 外部会变的事实必须行内带「（截至 YYYY-MM）」。

## 5. lint 规则速览（L1-L15）

frontmatter 完整性（L1/L2）、枚举合法（L3）、aliases≥2（L4）、时间戳序（L5）、updated 与 mtime 一致（L6，`--fix` 自动校正）、裸链接（L7）、边型与理由（L8/L15）、出处路径存在性（L9）、孤儿卡（L10）、同步冲突副本（L11）、已取代完整性（L12）、命名规范（L13）、长度预算（L14：MOC≤30 链，T2 卡≤40 行，HOME≤100 行）。

**维护三连**：每次改动后 `vault_lint.py` → `vault_catalog.py` → `vault_routemap.py`（+`vault_rings.py`），保持索引与卡永远一致。
