# 总体架构

*[English](../en/architecture.md)*

## 0. 一句话

第二大脑 = **判断层的持久化**：把 agent 在工作中做出的结论、决策、教训固化成卡片网络，用分层入口保证"任何知识 ≤8 次读取可达"，用自动回路保证网络随使用不断进化。

## 1. 双体大脑

| 体 | 内容 | 角色 |
|---|---|---|
| **蒸馏体**（本仓库的 `vault/`） | 数百张真卡 + 索引 | 判断知识：结论/边/理由/决策线。小而全对，每字都经过判断 |
| **全文体**（可选扩展） | 每项目的原文级全文镜像 + 机械关联 | 无限储存：出错可整夹重建，永不污染原项目文件 |

蒸馏体永远保持"一屏路由可达"；全文体按项目横向铺开、机械生成。本仓库 v1 只含蒸馏体框架；全文体构建器属文献/项目管线，规划为 v2。

## 2. 无壁垒：入口金字塔

从"常驻上下文"到"外部世界"，检索半径逐层放大，每层都有硬预算：

```
热核（开局必读，一屏：手工恒载区 + 度中心性 Top-20 知识原子）
 └─ 路由地图（第 0 跳，一屏：每项目 定位+关键词+承重卡 top-5，按 度+3×used 排序）
     └─ 题型路由表（按问题长相选入口，见 retrieval-protocol.md）
         └─ 枢纽现况节 / MOC 结论摘要节（第 1 跳，各 ≤10 行）
             └─ 卡（第 2 跳起，顺边走，每跳 ≤2 张）
                 └─ V 字跳（上位轴跨项目侧向发现）
                     └─ 索引层（总目录/邻接表/向量库，grep/语义一击定位）
                         └─ 外部检索（miss 升级：外找 → 必须回写成卡）
```

全程 **≤8 次文件读取封顶**——触顶未命中=路由失败，记入 `_检索失败日志.md`（miss 本身是改进网络的信号）。

## 3. 无限进化：五个自动回路

1. **入脑回路**：新东西出现 → `brain_intake.py` → 机械层产出卡骨架+候选边提名单（带证据词）→ 判断层确认落卡。一跑一确认，分钟级。
2. **miss 回路**：检索触底 → 外部检索核实 → 建卡 → **必须接进降链沿途卡**（挂边并写明"为什么当时没答上"）。每次失败让大脑长一块新组织。
3. **权重回路（Hebbian）**：真实调用 → `vault_touch.py` used+1 → 路由地图按 度+3×used 重排 → 常用知识浮到第 0 跳。收割钩子自动完成记账。
4. **挑战回路**：月度对高承重卡做对抗式质疑（这两卡矛盾吗？数据撑得住吗？）→ 产出冲突边/问题卡/降档提名。防僵化。
5. **巡检回路**：`vault_patrol.py`（月度）+ `brain_audit.py`（关联性体检）→ 晋升/降档/补边提案，人工处置。

## 4. 三环注意力视图（token 经济的核心机制之一）

卡有三种活性：**主线显性**（当前在办：项目进程/开放问题/最新内容，frontmatter 标 `活性: 显性`）、**辅助显性**（在办话题的过程记录，标 `活性: 辅助 YYYY-MM-DD`，**7 天保鲜自动降隐**）、**隐性**（缺省，不写字段）。

由此计算出三环（`vault_rings.py`，纯计算视图非静态标注）：

- **环 0**：显性卡 = 项目工作的入口；
- **环 1**：与环 0 直接相连的隐性卡 = 当前工作的支撑知识，经显性辐射一跳可达；
- **环 2**：其余全部 = 沉淀层（已结束话题+深层知识），入口=主题 MOC/编年层/总目录，不经显性辐射。

**撤一个显性标，它的支撑网自动沉入知识库，零搬运。** 项目工作检索只看环 0→环 1；历史/知识问题直接走环 2 框架入口——注意力和 token 都只花在该花的地方。

另有**环外知识平面**：`知识点-` 卡（维基式定义库）不参与环计算，环内卡出现该关键词时挂一条边（百科链接方式）。

## 5. 卡片网络的健康度量

`brain_audit.py` 十项机械检查：别名歧义 / 支持-冲突并存对 / 上位环 / 上位层级倒挂 / tier-度错配（高档低连、低档高连→反链升档）/ 星形卡（只连枢纽=没进真网络）/ 弱理由边 / 摘要合规 / 问题卡状态一致性 / 分布统计。健康的网络应做到：**零矛盾边、零上位环、零层级倒挂、零孤儿卡**。

## 6. 永不删除原则

价值不足=降档（tier 降级或转索引行），结论过时=`status: 已取代` + `取代者::` 边指向新卡，物理删除只发生在 `_回收站/` 且需人工批准。历史判断是资产：知道"当时为什么错"和知道"现在什么对"同样值钱。

## 7. 设计渊源
